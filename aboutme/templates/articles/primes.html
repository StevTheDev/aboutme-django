Finding Prime Numbers

It's a common lesson early on in one's programming career to calculate the first n prime numbers using nested loops. It's easy to understand how you could loop through lesser numbers looking for factors, and this makes for a great beginner exercise. The hidden reason for it being a great lesson however is that it exposes to us early on to the mathmatical reality of time-complexity.
--
Considering that we are creating a list of primes, and the Prime Number Theorem, we can assume that the nth prime of our list is approximately equal to n * logn(n). Using this method we see that in order to verify a candidate C as being prime, we must first do C^2 multiplication operations. For the final candidate this boils down to O(C^2 * len(C)^2). The total runtime of this function is approximately equal to the sum:
sum([ (i**2 * i.bit_length()**2) for i in range(2, (n * math.log2(n)))])
--
We can attempt to shortcut some of this runtime by exiting the loops as soon as possible, or by using clever mathmatics to preclude certain candidates as being prime. However for large n, even employing these methods will not result in a feasable runtime.
